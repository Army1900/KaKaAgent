# 通用任务规划

- name: 通用任务规划
- summary: 把模糊请求整理成目标、边界、阶段、产物和验收条件。
- permission: read
- triggers: 计划,规划,整理,需求,目标,下一步,方案

## 使用方式

当用户的输入还没有清晰执行路径时，先把会话整理成一个可推进任务：

1. 明确目标和不做什么。
2. 列出已知上下文和未知点。
3. 把任务拆成 3 到 6 个阶段。
4. 每个阶段写明输入、输出、权限和验收方式。
5. 如果需要读取本地文件，先建议绑定目录。

## 输出约束

- 不要把内部分类过程直接展示给用户。
- 不要承诺已经读取未授权的文件。
- 涉及写入、执行命令、联网、删除时必须进入审批。
