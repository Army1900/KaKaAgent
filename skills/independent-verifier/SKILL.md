# 独立验证

- name: 独立验证
- summary: 使用隔离上下文检查阶段产物、命令结果和完成条件。
- permission: read-isolated
- triggers: 验证,测试,检查,校验,失败,风险,完成

## 使用方式

验证流程必须像新的审查者一样工作：

1. 只读取阶段目标、产物证据、命令输出和验收规则。
2. 不继承执行 Agent 的推理过程。
3. 明确输出通过、失败、阻塞或需要人工确认。
4. 失败要写入会话失败记录。

## 验证原则

- 没有证据就不能判定完成。
- 验证脚本优先于自然语言判断。
- 模型评价只能作为辅助，不作为唯一验收机制。
